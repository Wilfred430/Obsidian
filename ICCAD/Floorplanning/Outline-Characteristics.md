# VLSI 平面規劃：Outline 種類與特性

在進行晶片設計規劃時，根據晶片邊界的限制程度，主要分為 Outline-free 與 Fixed-outline 兩大類。

## 1. Outline-free Floorplanning (傳統型)

這是最經典的平面規劃模式，主要目標是在滿足模組放置的前提下，盡可能縮小面積。

- **定義**：晶片的邊界（寬度與高度）是不固定的，可以隨內部模組的擺放位置而彈性變動。
- **優化目標**：
    - 最小化總面積 (Total Area)。
    - 最小化總線路長度 (Total Wirelength)。
- **缺點**：生成的晶片形狀可能非常奇特（過長或過寬），不符合實際後續封裝或頂層整合的需求。
- **常見算法**：早期的模擬退火 (SA)、B*-tree、Sequence Pair 等基礎應用。

## 2. Fixed-outline Floorplanning (現代主流)

隨著 SoC（系統單晶片）設計趨向複雜與層次化，這種模式已成為工業界標準。

- **定義**：晶片的寬度 (W) 與高度 (H) 是預先定義好的固定常數。所有模組必須被放置在這個矩形範圍內。
- **核心約束**：
    - **邊界約束**：模組座標 $(x, y)$ 加上其尺寸 $(w, h)$ 不得超出 $(W, H)$。
    - **零白空間 (Zero-whitespace)**：在某些高密度設計中，要求模組填滿整個 Outline。
- **適用場景**：
    - **層次化設計 (Hierarchical Design)**：當頂層已經分配好子模組的固定空間時。
    - **IP 整合**：將特定功能塊放入既有的晶片空白區域。
- **技術挑戰**：
    - **合法化難度高**：傳統算法很難在固定範圍內保證「無重疊 (Overlap-free)」。
    - **長寬比壓力**：當 Outline 比例與內部模組總面積太接近時，搜尋解答空間會變得極度困難。
- **解決方案**：
    - 基於 Slack 的移動：利用佈局餘裕來調整模組位置。
    - 解析式方法 (Analytical Methods)：如 QinFer 演算法，利用位能函數強制模組收斂在邊界內。

## 3. 兩者對比表

| 特性 | Outline-free (傳統) | Fixed-outline (現代) |
| :--- | :--- | :--- |
| **邊界限制** | 不固定，隨模組變動 | 預先定義，固定寬高 |
| **主要目標** | 縮小總面積 | 在限制內完成合法放置 (Legalization) |
| **白空間** | 越小越好 | 是一個既定的約束 (Constraint) |
| **設計風格** | 平坦化設計 (Flat) | 層次化設計 (Hierarchical) |
| **演算法難度** | 較低 | 極高 (NP-hard) |
