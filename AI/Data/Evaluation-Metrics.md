# 分類任務評估指標：Micro, Macro 與 Weighted 分數

在多分類任務中，當各類別的資料數量不平衡時，單一的準確度（Accuracy）無法反映真實表現。我們需要透過不同的平均方式來計算精確率（Precision）、召喚率（Recall）與 F1 分數。

## 基礎定義
在計算平均值之前，我們會先計算每一個類別各自的指標：
- Precision (精確率)：模型預測為正類中，真正為正類的比例。
- Recall (召喚率)：實際為正類中，被模型正確預測出的比例。
- F1-Score：精確率與召喚率的調和平均數。

## 三種平均方式比較

### 1. Macro Average (總體平均)
- 計算方式：不考慮類別樣本數量，直接將各類別的指標相加後除以類別總數。
- 公式：(類別1指標 + 類別2指標 + ... + 類別N指標) / N
- 特點：將所有類別「平等對待」。
- 適用場景：**本次 HW3 的首選**。當資料極度不平衡（如 15:1）時，Macro Average 能有效反映模型在少數類別上的表現。如果模型在少數類別表現很差，Macro 分數會大幅下降。

### 2. Micro Average (個體平均)
- 計算方式：將所有類別的 TP (真陽性)、FP (假陽性)、FN (假陰性) 全部加總在一起，視為一個大的二元分類任務來計算。
- 特點：受樣本數量多的類別影響極大。
- 適用場景：當你比較在乎「整體預測正確率」，且不在乎個別類別表現時。在多分類任務中，Micro Precision = Micro Recall = Micro F1 = Accuracy。

### 3. Weighted Average (加權平均)
- 計算方式：根據各類別的樣本數量（Support）進行加權計算。
- 公式：(類別1指標 * 類別1樣本數 + ... + 類別N指標 * 類別N樣本數) / 總樣本數
- 特點：考慮了類別不平衡，但比起 Macro，它更傾向於保護多數類別的表現。
- 適用場景：當你希望指標能反映出資料分佈的真實狀況時。

## 總結對比

| 指標類型 | 對不平衡資料的敏感度 | 核心特點 |
| :--- | :--- | :--- |
| Macro | 極高 | 視所有類別同等重要，強迫模型學習少數類別。 |
| Micro | 低 | 重視整體正確樣本數，容易被多數類別掩蓋失敗。 |
| Weighted | 中 | 根據類別出現頻率給予權重，介於兩者之間。 |

## 在 HW3 中的實踐意義
由於 HW3 的資料比例為 15:1，模型很容易為了刷高 Accuracy 而放棄學習少數類別。使用 **Macro F1** 是為了確保你必須針對那 1/16 的少數樣本進行資料增強或 Prompt 優化，否則你的 Kaggle 排名將無法提升。
