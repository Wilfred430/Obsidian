# 🤖 LLM 指令與規範 (System Instructions)

> [!info] **地位**：本文件是 LLM 在操作此 Obsidian 倉庫時的最高準則，優先於通用設定。

## 1. 核心操作原則
- **雙重記錄**：每次知識變動必須記錄於 `log.md`。
- **結構考量**：若發生重大目錄重構或系統升級，必須同步更新 `README.md` 與 `index.md`。
- **三層限制**：資料夾路徑嚴格限制在最多三層（例如：`Domain/Category/Node.md`）。

## 2. 寫作風格
- **禁止表情符號**：Markdown 檔案內絕對不使用 emoji。
- **邏輯至上**：優先使用 Callouts (`[!info]`, `[!danger]`, `[!success]`) 進行視覺化分類。
- **原子化筆記**：鼓勵建立細小的知識節點（Atomic Notes），並透過 `[[Link]]` 進行網狀連結。

## 3. 命名規範
- **精確性**：使用技術關鍵字作為檔名，避免模糊語句。
- **英文優先 (路徑)**：資料夾路徑建議使用英文，筆記內容與標題使用中文。
