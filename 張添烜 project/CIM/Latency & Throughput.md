
## Latency (延遲)
- 定義：系統從輸入到輸出之間的時間差
- 強調 **反應速度**
- 單位：時間 (ns, ms, 秒)
- 舉例：
  - 記憶體存取延遲
  - 網頁載入等待時間
  - 遊戲操作反應時間

## Throughput (吞吐量)
- 定義：系統在單位時間內能處理的工作量
- 強調 **處理效率**
- 單位：操作數/秒、位元率 (bps)、任務/秒
- 舉例：
  - 網路每秒可傳輸的資料量
  - CPU 每秒可完成的指令數
  - GPU 每秒可處理的影像幀數

## 對照表
| 指標       | Latency (延遲) | Throughput (吞吐量) |
|------------|----------------|---------------------|
| 著重面向   | 反應時間       | 處理效率            |
| 測量單位   | 時間 (ns, ms)  | 工作量/秒、bps      |
| 使用情境   | 記憶體存取、網路反應 | 系統效能、資料傳輸 |
| 關鍵問題   | 等多久？       | 一次能處理多少？    |

## 簡單比喻
- **Latency**：等公車的時間  
- **Throughput**：一輛公車能載多少人
