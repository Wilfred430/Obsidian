# 中斷機制核心：ISR、IST、ISTP 與 ISFP

## TL;DR 核心摘要
這四個機制共同構成了 [[CPU]] 從接收硬體中斷信號到執行對應處理程式的「位址檢索與跳轉鏈結」，其中 **ISTP** 定位表格，**IST** 存儲結構，**ISFP** 提供跳轉指令，最終引導至 **ISR** 執行任務。

## 概念隱喻：找人處理緊急事件（通訊錄）
為了快速處理突發狀況，你建立了一套緊急應變系統：
- **ISTP (指針)**：你腦中記得「應急通訊錄」放在書架的哪個位置（基底位址）。
- **IST (通訊錄)**：那本放在書架上的「應急通訊錄總表」，裡面按編號排好了各類事故。
- **ISFP (索引分頁)**：翻開通訊錄後，某一頁上面寫著的「簡短動作指引」（例如：請轉撥分機 119 或 翻到手冊第 50 頁）。
- **ISR (處理專員)**：根據指引找到的「專業救援人員」，他會帶領你完成整套滅火與撤離流程（詳細處理程式）。

## 名詞定義清單
1. **ISTP (Interrupt Service Table Pointer)**：
    - **全名**：中斷服務表指針。
    - **定義**：一個底層 [[暫存器]]，存放 [[中斷向量表]]（IST）的起始基底位址。在 [[TMS320C6000]] 架構中，其低位通常有對齊限制（如 1KB 對齊）。
2. **IST (Interrupt Service Table)**：
    - **全名**：中斷服務表。
    - **定義**：存放在 [[記憶體]] 中的數據結構，由多個中斷向量空間組成，每個中斷通道對應一個特定的偏移位置。
3. **ISFP (Interrupt Service Fetch Packet)**：
    - **全名**：中斷服務取指包。
    - **定義**：IST 中分配給單個中斷的固定大小空間（例如 32 Bytes）。它通常不存放完整的程式碼，而是存放一組起始指令（如跳轉指令 B），作為進入 ISR 的門戶。
4. **ISR (Interrupt Service Routine)**：
    - **全名**：中斷服務程式。
    - **定義**：工程師編寫的實際處理函式，負責執行特定的演算法或 I/O 操作，結束後需返回原程式。

## 運作流程
當硬體觸發中斷事件（如定時器到期）時，[[CPU]] 會依序執行以下步驟：
1. **定位基底**：硬體讀取 **[[ISTP]]** 暫存器，得知 [[中斷向量表]] 在記憶體中的首位址。
2. **計算偏移**：根據中斷編號（如 INT4, INT5），計算出該中斷對應的 **[[IST]]** 項目偏移量。
3. **擷取入口**：CPU 從計算出的位址抓取一個 **[[ISFP]]**（取指包）。
4. **執行跳轉**：執行 ISFP 內的 [[組合語言]] 指令（通常是 B 指令），將程式計數器（PC）導向真正的 **[[ISR]]** 起始點。
5. **程式處理**：進入 ISR 執行業務邏輯，並在完成後透過中斷返回指令（如 B IRP）回到被中斷前的執行點。
